Find documents by key

First we login to CouchDB and generate the auth_session token for making requests.

hash = Couchdb.login(username = 'samson',password ='sam123') 
auth_session =  hash["AuthSession"]

To find documents by email key (example: to find all contacts with email = ‘nancy@mail.com’)

Couchdb.find_by({:database => 'contacts', :email => 'nancy@mail.com'} , auth_session)  
 
# => [{"_id"=>"Nancy", "_rev"=>"1-d15a83d2a23b495c19df2595b636ecc8", "firstname"=>"Nancy", "lastname"=>"Lee",
#  "phone"=>"347-808-3734", "email"=>"nancy@mail.com", "gender"=>"female"}]

To find all contacts with lastname = ‘Smith’

Couchdb.find_by({:database => 'contacts', :lastname => 'Smith'},auth_session)  
 
# => [{"_id"=>"john", "_rev"=>"5-642689e0a50843d6fa508159a01b4fd4", "firstname"=>"John", "lastname"=>"Smith",
#   "email"=>"john@mail.com", "gender"=>"male"}]

To find all female contacts

Couchdb.find_by({:database => 'contacts', :gender => 'female'},auth_session) 
 
# => [{"_id"=>"Nancy", "_rev"=>"1-d15a83d2a23b495c19df2595b636ecc8", "firstname"=>"Nancy", "lastname"=>"Lee",
#  "phone"=>"347-808-3734", "email"=>"nancy@mail.com", "gender"=>"female"}]

If you are running an admin party (i.e you have no admins in CouchDB), simply use the same methods, with no auth_session parameter.

Couchdb.find_by( :database => 'contacts', :gender => 'female')

Options
A set of options can be used to manipulate the results.

limits
To limit the results. Example to return the first 20 female contacts:

options = {:limit => 20}
Couchdb.find_by({:database => 'contacts', :gender => 'female'},auth_session, options)

skip
To skip the first 10 female contacts and return 20:

options = {:skip => 10,:limit => 20}
Couchdb.find_by({:database => 'contacts', :gender => 'female'},auth_session, options)

Note that the skip option is considered a relatively expensive CouchDB operations because the rows are still processed and skipped.

To skip the first 20 female contacts and return all remaining contacts:

options = {:skip => 20}
Couchdb.find_by({:database => 'contacts', :gender => 'female'},auth_session, options)

descending
Couchdb returns results in ascending order by default, to change it to descending order:

options = {:descending => true}
Couchdb.find_by({:database => 'contacts', :gender => 'female'},auth_session, options)

startkey & endkey
To return the results within a range. For example in an employee database, to return all employees with the salary range 53000 to 99000

options = {:startkey => "53000", :endkey => "99000"}
Couchdb.find_by({:database => 'employees', :salary => ''},auth_session, options)

To return the first 20 employees with salary between 53000 and 99000.

options = {:startkey => "53000", :endkey => "99000",:limit => 20}
Couchdb.find_by({:database => 'employees', :salary => ''},auth_session, options)

To return all employees with salary 99000 or less:

options = {:endkey => "99000"}
Couchdb.find_by({:database => 'employees', :salary => ''},auth_session, options)

How it works
The method

Couchdb.find_by({:database => 'contacts', :email => 'nancy@mail.com'}, auth_session)

sends a

GET http://127.0.0.1:5984/contacts/_design/email_finder/_view/find_by_email?key="nancy@mail.com"

This returns all contacts with email = ‘nancy@mail.com’. Leanback will parse the native JSON results to return only the data values. If the design_document (_design/email_finder) and the view (find_by_email) doesn’t exist, Leanback will create it in the database. See the source for the generated design_document;

{
  "_id": "_design/email_finder",
  "_rev": "7-da7f3c0bf183f4a36a82013bd0ea6537",
  "language": "javascript",
  "views": {
      "find_by_email": {
          "map": "function(doc){
             if(doc.email)          
                 emit(doc.email,doc); 
                }"
      }
  }
}

If you ran

Couchdb.find_by({:database => 'contacts', :lastname => 'Smith'},auth_session)  
Couchdb.find_by({:database => 'contacts', :gender => 'female'},auth_session)

you will notice the following design_documents have been added to the database

_design/lastname_finder
_design/gender_finder

Usage
You can do

 docs = Couchdb.find_by({:database => 'contacts', :email => 'nancy@mail.com'},auth_session)  
    docs.each do |d| 
	puts d["_rev"]
	puts d["_id"]    
	puts d["firstname"]
	puts d["lastname"]
	puts d["email"] 
	puts d["phone"]
   end
 
# => rev1-d15a83d2a23b495c19df2595b636ecc8
#    Nancy
#    Nancy
#    Lee
#    nancy@mail.com
#    347-808-3734
