Find document by multiple Keys

First we login to CouchDB and generate the auth_session token for making requests.

hash = Couchdb.login(username = 'samson',password ='sam123') 
auth_session =  hash["AuthSession"]

To retrieve all documents in the friends database with gender = male and age = 34

 keys = {:gender => 'male',:age => '34'}
 Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session)

To retrieve all documents in the friends database with gender = male, age = 34 and lastname = smith

 keys = {:gender => 'male',:age => '34', :lastname => 'smith'}
 Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session)

If you are running an admin party (i.e you have no admins in CouchDB), simply use the same methods, with no auth_session parameter.

keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys})

Options
A set of options can be used to manipulate the results.

limits
To limit the results. Example to return the first 20 documents with age=34 and gender = male:

options = {:limit => 20}
keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session, options)

For admin party,

options = {:limit => 20}
keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys},"", options)

skip
To skip the first 10 results and return the next 20:

options = {:skip => 10,:limit => 20}
keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session, options)

Note that the skip option is considered a relatively expensive CouchDB operation because the rows are still processed and skipped.

To skip the first 20 results and return the rest:

options = {:skip => 20}
keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session, options)

descending
Couchdb returns results in ascending order by default, to change it to descending order:

options = {:descending => true}
keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session, options)

startkey & endkey
startkey & endkey don’t work with find_by_keys() because find_by_keys already uses them. See the next section for details.

How it works
The method

keys = {:gender => 'male',:age => '34'}
Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session)

sends a

GET http://127.0.0.1:5984/friends/_design/gender_age_keys_finder/_view/find_by_keys_gender_age?startkey=["male","34"]&endkey=["male","34"]

This returns all friends with gender = male and age = 34. Leanback will parse the native JSON results to return only the data. If the design_document (_design/gender_age_keys_finder) and the view (find_by_keys_gender_age) doesn’t exist, Leanback will create it in the database. See the source for the generated design_document;

{
   "_id": "_design/gender_age_keys_finder",
   "_rev": "1-92674ebbb4bca65add937f764b37623d",
   "language": "javascript",
   "views": {
       "find_by_keys_gender_age": {
           "map": "function(doc){ if(doc.gender && doc.age) emit([doc.gender,doc.age],doc);}"
       }
   }
}

Usage
You can do

 keys = {:gender => 'male',:age => '34'}
 docs = Couchdb.find_by_keys({:database => 'friends', :keys => keys},auth_session)   
 docs.each do |d| 
   puts d["_rev"]
   puts d["_id"]    
   puts d["firstname"]
   puts d["lastname"]
   puts d["age"] 
   puts d["gender"]
  end
